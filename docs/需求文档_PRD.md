## 1. 产品概述

### 1.1 产品背景

我国有大量轻度心智障碍人士（如高功能孤独症谱系人士、轻度智力发育迟缓者）具备参与社会劳动的能力和意愿。他们通常可以胜任保洁、后厨帮工、图书整理、超市理货、咖啡店服务员等结构化程度较高的岗位。然而，由于在记忆、执行、社交沟通方面存在一定困难，他们在求职、适应岗位、维持稳定就业等环节面临诸多挑战。

与此同时，就业辅导员人手严重不足，家长长期焦虑，雇主和同事虽有包容意愿却缺乏有效方法。当前市场上缺少一款专门针对这一群体的、极简易用的数字化职场支持工具。

### 1.2 产品愿景与目标

**愿景：** 让每一位轻度心智障碍人士都能获得持续、可及的职场支持，实现稳定就业和自我价值。

**产品目标：**

| 目标维度 | 具体描述 |
|----------|----------|
| 员工独立度 | 通过视觉化任务引导，帮助员工减少对人工陪同的依赖，独立完成工作流程 |
| 支持效率 | 通过远程协助和自动预警机制，使每位辅导员可同时支持更多员工 |
| 家长安心 | 提供透明的数据反馈渠道，让家长实时了解孩子的工作状态 |
| 雇主信心 | 降低雇主对雇佣心智障碍员工的顾虑，促进包容性就业 |

### 1.3 产品形态

| 端 | 形态 | 目标用户 | 设计原则 |
|----|------|----------|----------|
| 员工端 | 移动端 APP（iOS / Android） | 心智障碍员工 | 极简、大图标、高对比度、语音辅助，强调"用起来"而非"学起来" |
| 辅导员端 | Web 网页端（响应式） | 就业辅导员、家长 | 功能完整、操作高效、数据可视化 |
| 雇主端 | Web 网页端（有限查看） | 雇主/企业管理者 | 仅可查看本企业相关员工的任务完成概况 |

### 1.4 术语定义

| 术语 | 说明 |
|------|------|
| 员工 | 使用员工端 APP 的心智障碍人士 |
| 辅导员 | 就业辅导员，负责岗位开发、职前培训和在职跟随支持的专业人员 |
| 家长 | 员工的家长或法定监护人 |
| 雇主 | 雇用心智障碍员工的企业或组织 |
| 任务模板 | 辅导员创建的、可复用的工作任务拆解方案 |
| 排班 | 辅导员为员工安排的当日工作任务列表 |
| 情绪温度计 | 通过员工主动表达和系统自动检测，量化评估员工当前情绪状态的机制 |
| 卡顿热点 | 员工在任务执行过程中停留时间异常长的步骤 |
| 隐形辅导员 | 远程协助功能，员工可一键发起，辅导员远程提供图文语音指导 |

### 2.1 核心用户（员工端）

| 年龄 | 22 岁 |
| 岗位 | 超市理货员 |
| 特征 | 对固定流程适应良好，但面对突发变化（如货架调整、临时换岗）容易焦虑；语言表达偏书面化，口语沟通有困难；对视觉信息敏感，能通过图片快速理解指令 |
| 痛点 | 记不住临时增加的新任务；不知道如何向同事求助；被顾客催促时会慌乱 |
| 期望 | 有明确的步骤指引，不用反复问别人；遇到困难时能快速获得帮助 |

**画像 B：小王（轻度智力发育迟缓）**

| 维度 | 描述 |
|------|------|

| 期望 | 有简单的方法了解员工需要什么帮助；员工能更独立地完成工作 |

### 2.3 用户关系模型

```
辅导员（管理员）
  ├── 管理多名员工（创建任务、排班、查看数据、接收预警）
  ├── 可邀请家长关联员工账号（家长获得只读权限）
  └── 可关联雇主账号（雇主获得有限查看权限）

家长（只读观察者）
  └── 查看自己孩子的任务完成情况、情绪记录、预警通知

雇主（有限查看者）
  └── 查看本企业相关员工的任务完成概况
```

## 3. 功能需求详述

### 3.1 用户注册与登录

#### 3.1.1 功能描述

系统支持三类角色（辅导员、家长、员工）的注册与登录。员工账号由辅导员代为创建，以降低员工自行操作的门槛。

#### 3.1.2 用户故事

| 编号 | 角色 | 故事 |
|------|------|------|
| US-REG-01 | 辅导员 | 作为辅导员，我希望通过手机号注册账号，以便快速开始使用系统 |
| US-REG-02 | 辅导员 | 作为辅导员，我希望能在系统中为每位员工创建账号（设置姓名、头像、岗位信息），以便员工直接使用 |
| US-REG-03 | 辅导员 | 作为辅导员，我希望能邀请家长绑定员工的账号，以便家长可以查看孩子的工作情况 |
| US-REG-04 | 员工 | 作为员工，我希望打开 APP 后直接进入主界面，不需要每次都输入密码 |
| US-REG-05 | 家长 | 作为家长，我希望通过辅导员发送的邀请链接完成注册并绑定孩子，以便开始查看信息 |

#### 3.1.3 交互流程

**辅导员注册流程：**

```
输入手机号 → 获取短信验证码 → 输入验证码 → 设置密码 → 填写基本信息（姓名、所属机构）→ 注册成功 → 进入辅导员端
```

**员工账号创建流程：**

```
辅导员登录 → 进入"员工管理" → 点击"添加员工" → 填写员工信息（姓名、昵称、头像、岗位、工作地点）→ 生成员工账号 → 辅导员将账号信息配置到员工手机上
```

**员工登录流程：**

```
员工打开 APP → 自动登录（首次需辅导员配置设备绑定）→ 进入今日任务主界面
```

**家长绑定流程：**

```
辅导员生成邀请链接/二维码 → 发送给家长 → 家长打开链接 → 手机号注册 → 自动绑定对应员工 → 进入家长端查看页面
```

#### 3.1.4 验收标准

| 编号 | 验收标准 |
|------|----------|
| AC-REG-01 | 辅导员可通过手机号 + 短信验证码完成注册，验证码 60 秒内有效 |
| AC-REG-02 | 辅导员创建员工账号后，员工使用对应设备打开 APP 可自动登录，无需手动输入账号密码 |
| AC-REG-03 | 员工端支持设备绑定机制，一台设备绑定一个员工账号，更换设备需辅导员授权 |
| AC-REG-04 | 家长通过邀请链接注册后，仅能看到自己孩子的数据，无法看到其他员工信息 |
| AC-REG-05 | 注册和登录流程在正常网络条件下均可在 3 秒内完成 |

### 3.2 权限管理

#### 3.2.1 功能描述

系统采用基于角色的访问控制（RBAC），支持辅导员、家长、雇主三种角色，各角色拥有不同的数据访问和操作权限。

#### 3.2.2 用户故事

| 编号 | 角色 | 故事 |
|------|------|------|
| US-AUTH-01 | 辅导员 | 作为辅导员，我希望可以管理多名员工的全部数据和配置，以便高效开展支持工作 |
| US-AUTH-02 | 辅导员 | 作为辅导员，我希望可以将特定员工的查看权限授权给家长，并控制家长可见的数据范围 |
| US-AUTH-03 | 家长 | 作为家长，我希望只能查看自己孩子的数据，且无法修改任何配置 |
| US-AUTH-04 | 辅导员 | 作为辅导员，我希望可以邀请同事共同管理同一批员工，并设置不同的管理权限 |
| US-AUTH-05 | 系统管理员 | 作为系统管理员，我希望可以管理辅导员的账号状态（启用/禁用），以保障系统安全 |

#### 3.2.3 权限矩阵

| 功能/数据 | 辅导员（主管理） | 辅导员（协管） | 家长 | 员工 |
|-----------|:---:|:---:|:---:|:---:|
| 创建/编辑任务模板 | 可 | 可 | 不可 | 不可 |
| 排班管理 | 可 | 可 | 不可 | 不可 |
| 查看任务完成详情 | 可 | 可 | 仅限关联员工 | 仅限自己 |
| 查看情绪记录 | 可 | 可 | 仅限关联员工 | 不可 |
| 接收预警通知 | 可 | 可 | 仅限关联员工 | 不可 |
| 远程协助（隐形辅导员） | 可 | 可 | 不可 | 发起方 |
| 修改员工个人信息 | 可 | 仅限所管员工 | 不可 | 不可 |
| 管理家长绑定 | 可 | 不可 | 不可 | 不可 |
| 查看数据统计报表 | 可 | 仅限所管员工 | 仅限关联员工概览 | 不可 |
| 自定义沟通板内容 | 可 | 可 | 不可 | 不可 |

#### 3.2.4 验收标准

| 编号 | 验收标准 |
|------|----------|
| AC-AUTH-01 | 不同角色登录后看到的菜单和功能严格按权限矩阵控制 |
| AC-AUTH-02 | 家长账号只能查看被授权绑定的员工数据，无法通过 URL 篡改访问其他员工数据 |
| AC-AUTH-03 | 协管辅导员只能操作被分配的员工，无法越权管理其他辅导员负责的员工 |
| AC-AUTH-04 | 辅导员可随时解除家长的绑定关系 |

### 3.3 核心功能模块一：任务拆解与视觉化流程导航

#### 3.3.1 功能描述

本模块是产品的核心功能，支持辅导员将一项工作任务拆解为多个可视化步骤，员工通过极简的 APP 界面逐步执行任务。系统通过大图标、高对比度色彩、语音提示和震动反馈等多模态方式引导员工完成任务。

#### 3.3.2 用户故事

| 编号 | 角色 | 故事 |
|------|------|------|
| US-TASK-01 | 辅导员 | 作为辅导员，我希望可以上传实拍照片或短视频，将一项工作拆解为多个步骤并为每步配图，以便员工能直观理解每个操作 |
| US-TASK-02 | 辅导员 | 作为辅导员，我希望可以将已创建的任务模板分配给员工的排班中，以便员工当天能看到并执行 |
| US-TASK-03 | 辅导员 | 作为辅导员，我希望可以为关键步骤设置特殊提醒方式（如震动、语音提示），以便员工不会遗漏重要环节 |
| US-TASK-04 | 辅导员 | 作为辅导员，我希望可以复用和修改已有任务模板，以减少重复创建的工作量 |
| US-TASK-05 | 员工 | 作为员工，我希望打开 APP 后直接看到今天要做的任务，不需要额外操作 |
| US-TASK-06 | 员工 | 作为员工，我希望进入任务后能看到清晰的步骤指引，完成一步后自动进入下一步 |
| US-TASK-07 | 员工 | 作为员工，我希望在关键步骤能听到语音提示和感受到手机震动，以帮助我集中注意力 |
| US-TASK-08 | 员工 | 作为员工，我希望可以返回上一步重新查看，以防我操作错了 |
| US-TASK-09 | 辅导员 | 作为辅导员，我希望可以查看每位员工每个任务的完成进度和每步耗时，以便识别需要额外支持的环节 |

#### 3.3.3 交互流程

**辅导员创建任务模板流程：**

```
进入"任务模板管理" → 点击"新建模板" → 输入任务名称和描述
→ 上传封面图片 → 点击"添加步骤"
→ 步骤 1：输入步骤名称 → 上传步骤配图/短视频 → 输入步骤说明文字
→ 设置是否为关键步骤（开启震动/语音提醒）→ 设置预计完成时长
→ 点击"添加下一步" → 重复上述操作
→ 全部步骤添加完毕 → 点击"保存模板"
→ 可选择"立即排班"将任务分配给指定员工
```

**辅导员排班流程：**

```
进入"排班管理" → 选择日期和员工 → 从任务模板库中选择任务 → 设置执行顺序和时间
→ 可添加备注（如"今天货架区域调整，注意新位置"）→ 确认发布
→ 员工端自动同步更新
```

**员工执行任务流程：**

```
打开 APP → 自动显示今日任务列表（大图标卡片形式）
→ 点击任务卡片 → 进入步骤引导界面
→ 显示当前步骤：大图片 + 简短文字说明
→ 完成操作后点击绿色大勾按钮 → 自动进入下一步
→ 关键步骤：播放语音提示 + 手机震动
→ 所有步骤完成 → 显示"太棒了！"庆祝动画 → 返回任务列表
```

**员工返回上一步流程：**

```
在步骤引导界面 → 点击左上角"返回上一步"图标（蓝色箭头）
→ 回到上一步骤 → 可重新查看 → 点击绿色大勾继续
```

**步骤返回状态说明：**

- 员工返回已完成的步骤时，该步骤标记为"已完成"状态（显示绿色勾），但内容仍可查看
- 员工在已完成的步骤上点击绿色大勾不会重复计数，仅作为"我已查看，继续下一步"的操作
- 返回操作不改变步骤的原始完成时间戳
- 辅导员端查看执行记录时，可看到员工是否有返回查看的行为（记录返回次数，但不影响步骤完成状态）

### 3.4 核心功能模块二：沟通板与情绪温度计

| US-COM-04 | 辅导员 | 作为辅导员，我希望当员工多次表达消极情绪或长时间卡在某一步骤时，系统能自动通知我 |

**常用语点击发声流程：**

```
员工点击底部导航栏"沟通板"图标 → 进入沟通板界面
→ 界面显示 6-8 个大图标按钮（如"我需要帮助""我去洗手间""我没听懂""我完成了"等）
→ 员工点击某个按钮 → 手机以清晰语音播报对应语句（音量自动调至合适水平）
→ 同时在屏幕上以大字显示该语句，方便展示给对方看
```

**情绪表达流程：**

```
员工点击"我的心情"入口 → 进入情绪选择界面
→ 显示 5-6 个情绪卡片（开心、平静、紧张、难过、生气、不舒服），每个卡片配有形象插图
→ 员工点击当前情绪 → 系统记录时间和情绪类型

```

**情绪温度计自动预警流程：**

```
系统持续监测以下信号：
  1. 员工在 10 分钟内连续 3 次选择消极情绪卡片
  2. 员工在单个任务步骤上停留超过预计时长的 3 倍
  3. 员工连续 2 次点击"我需要帮助"
→ 触发任一条件 → 系统自动向辅导员和家长发送"支持请求"通知
→ 通知内容包括：员工姓名、触发原因、当前所在任务和步骤、持续时间
```

| US-ENV-04 | 员工 | 作为员工，我希望遇到不会做的事情时，可以拍照发给辅导员，辅导员帮我标注后发回来 |

```
辅导员在排班时设置上班时间、下班时间和休息时间段
→ 到达提醒时间前 5 分钟 → 员工端弹出全屏提醒卡片（大图标 + 文字 + 语音播报）
→ "还有 5 分钟就要休息啦！"（休息提醒）或"还有 5 分钟就要下班啦！"（下班提醒）
→ 到达精确时间 → 再次弹出提醒 + 震动

```

```
  - "我不舒服，需要去医务室"
  - "我需要打电话给我的家人"
  - "这里不安全，请帮帮我"
  - "我需要安静一下"
  - "我听不懂，请再说一次"
→ 员工点击对应卡片 → 全屏显示该语句（超大字体，高对比度）
→ 同时语音播报该语句 → 方便展示给管理者或同事看
→ 点击"关闭"返回
```

**"隐形辅导员"远程协助流程：**

```
员工在任务执行界面或沟通板点击"求助辅导员"按钮
→ 选择"拍照求助" → 打开相机拍照（或从相册选择）
→ 照片发送给辅导员 → 员工端显示"已发送，请稍等..."
→ 辅导员端收到请求 → 打开照片 → 使用圈画工具标注（画圈、箭头、文字）
→ 可选：录制语音说明（如"小林，看红色圈的地方，把箱子放到这里"）
→ 发送回员工端 → 员工端收到标注后的图片 + 语音播放
→ 员工查看后点击"我明白了"或"还是不懂"（后者再次通知辅导员）
```

### 3.8 离线模式

#### 3.8.1 功能描述

#### 3.8.2 用户故事

| 编号 | 角色 | 故事 |
|------|------|------|

| US-OFF-03 | 辅导员 | 作为辅导员，我希望员工恢复网络后，能自动同步离线期间的操作数据 |

#### 3.8.3 离线功能支持范围

| 功能 | 离线可用 | 说明 |
|------|:---:|------|
| 查看今日任务列表 | 是 | 需提前联网同步 |
| 执行任务步骤（查看图片、点击完成） | 是 | 步骤图片和视频需提前缓存 |
| 语音提示播放 | 是 | 语音文件需提前缓存 |
| 沟通板常用语点击发声 | 是 | TTS 语音需提前缓存为音频文件 |
| 情绪表达记录 | 是 | 本地暂存，联网后自动上传 |

| 应急卡片展示 | 是 | 纯本地功能 |
| 远程协助 | 否 | 需网络支持 |
| 发送预警通知 | 否 | 本地记录，联网后自动补发 |
| 排班更新同步 | 否 | 需网络支持 |

#### 3.8.4 数据同步策略

```
联网状态 → 自动在后台下载/更新以下数据：
  - 当日及次日排班任务（含步骤图片、视频、语音文件）
  - 沟通板配置内容（含预生成的语音文件）
  - 应急卡片配置
  - 提醒时间配置

断网状态 → 所有操作记录保存在本地数据库：
  - 任务步骤完成记录（含完成时间戳）
  - 情绪表达记录
  - 沟通板使用记录

恢复网络 → 自动执行同步：
  - 上传本地暂存的操作记录
  - 补发离线期间触发的预警通知
  - 下载最新的排班和配置更新
  - 同步完成后在状态栏显示"数据已同步"
```

#### 3.8.5 验收标准

| 编号 | 验收标准 |
|------|----------|
| AC-OFF-01 | 员工端在联网时自动缓存当日及次日全部任务数据（图片、视频、语音），无需用户手动操作 |
| AC-OFF-02 | 离线状态下，员工可正常查看任务步骤、点击完成、使用沟通板和应急卡片 |
| AC-OFF-03 | 离线期间的操作数据在恢复网络后 30 秒内自动完成同步 |
| AC-OFF-04 | 离线状态下，APP 顶部状态栏显示"离线模式"标识，但不影响正常使用 |
| AC-OFF-05 | 离线缓存数据在任务过期后自动清理，避免占用过多设备存储空间 |

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 员工端启动时间 | 冷启动不超过 3 秒（中端手机，如骁龙 680 及以上） |
| 页面切换时间 | 不超过 0.5 秒 |
| 任务步骤图片加载 | 不超过 1 秒（已缓存情况下不超过 0.3 秒） |
| 语音播报延迟 | 点击后不超过 0.5 秒开始播放 |
| 辅导员端页面加载 | 不超过 3 秒 |
| 数据报表生成 | 不超过 5 秒 |
| API 响应时间 | 95% 的请求在 500ms 内响应 |
| 系统可用性 | 99.5%（每月非计划停机不超过 3.6 小时） |
| 并发支持 | 支持 5000 名员工同时在线执行任务 |

### 4.2 安全需求

| 需求项 | 说明 |
|--------|------|
| 数据传输加密 | 所有数据传输使用 TLS 1.2+ 加密 |
| 数据存储加密 | 敏感数据（如手机号）在数据库中加密存储 |
| 身份认证 | 辅导员端和家长端采用 JWT + Refresh Token 机制，Token 有效期 2 小时 |
| 员工端设备绑定 | 员工账号与设备绑定，更换设备需辅导员在后台操作授权 |
| 接口鉴权 | 所有 API 接口基于角色进行权限校验，防止越权访问 |
| 数据隔离 | 不同辅导员管理的员工数据严格隔离，协管辅导员仅能访问被授权的数据 |
| 操作日志 | 记录辅导员和管理员的关键操作（创建、修改、删除、导出），日志保留 180 天 |
| 隐私保护 | 员工的情绪记录、卡顿数据等敏感信息仅辅导员和家长可见，不对雇主开放 |
| 图片/视频安全 | 上传的图片和视频经过内容安全审核，防止违规内容 |

### 4.3 可用性需求

| 需求项 | 说明 |
|--------|------|
| 员工端学习成本 | 新员工在辅导员引导下，首次使用即可完成基本操作，无需专门培训 |
| 操作容错 | 员工端所有关键操作（如删除、跳过步骤）需二次确认，防止误操作 |
| 错误恢复 | 网络异常时给出友好提示而非报错页面；操作中断后可从断点继续 |
| 辅导员端效率 | 辅导员创建一个包含 10 个步骤的任务模板，操作时间不超过 10 分钟 |
| 帮助引导 | 辅导员端提供操作指引和帮助文档；员工端通过图标和颜色直觉引导 |

### 4.4 兼容性需求

| 平台 | 最低版本要求 |
|------|-------------|
| Android | Android 8.0（API Level 26）及以上 |
| iOS | iOS 13.0 及以上 |
| 辅导员端浏览器 | Chrome 90+、Firefox 88+、Safari 14+、Edge 90+ |
| 屏幕适配 | 员工端支持 4.7 英寸至 6.9 英寸屏幕；辅导员端支持 1280px 至 2560px 宽度 |

### 4.5 无障碍设计要求

本产品面向心智障碍用户，无障碍设计是核心设计原则而非附加要求。

| 需求项 | 具体要求 |
|--------|----------|
| 视觉无障碍 | 全局高对比度模式（默认开启），对比度不低于 4.5:1；支持大字体模式（默认大字体）；避免使用颜色作为唯一信息传达方式 |
| 认知无障碍 | 界面元素数量最小化，每个页面核心操作不超过 3 个；使用一致的布局和交互模式；避免使用抽象图标，所有图标配文字标签 |
| 听觉无障碍 | 所有语音提示同时提供视觉反馈（文字 + 动画）；支持调节语音播报速度（0.8x / 1.0x / 1.2x） |
| 运动无障碍 | 主要操作按钮尺寸不小于 80x80dp；按钮间距不小于 16dp；支持长按操作的替代方案（如点击确认） |
| 交互一致性 | 相同功能在不同页面使用相同的图标、颜色和位置；导航结构固定不变 |
| 信息分步呈现 | 每次只呈现一个步骤或一个决策，避免信息过载 |
| 正向反馈 | 员工完成操作后提供即时的正向视觉/听觉反馈（如勾选动画、简短鼓励语音） |

## 7. 风险与约束

### 7.1 风险分析

| 风险 | 等级 | 影响 | 应对策略 |
|------|:---:|------|----------|
| 员工对智能手机操作不熟练 | 高 | 产品无法被目标用户有效使用 | 与辅导员紧密配合，在入职培训阶段完成 APP 操作引导；界面设计极致简化 |
| 网络环境不稳定 | 中 | 离线场景下部分功能不可用 | 实现完善的离线模式，核心功能离线可用 |
| 员工隐私数据泄露 | 高 | 员工和家长的信任受损，法律风险 | 严格的数据加密和权限控制；定期安全审计；符合《个人信息保护法》要求 |
| 辅导员推广意愿不足 | 中 | 产品 uptake 速度慢 | 降低辅导员使用成本（减少重复操作）；通过数据报表体现工作价值 |
| 员工过度依赖 APP | 低 | 离开 APP 后无法独立工作 | 设计"逐步淡出"机制，随着员工熟练度提升，逐步减少提示频率 |
| TTS 语音播报质量不佳 | 中 | 员工听不清或听不懂 | 使用高质量中文 TTS 引擎；支持自定义录音替代 TTS |
| 雇主配合度低 | 中 | 员工使用 APP 受到限制 | 向雇主提供简短的说明材料，强调 APP 对提升工作效率的积极作用 |

### 7.2 约束条件

| 约束 | 说明 |
|------|------|
| 法律合规 | 产品需符合《中华人民共和国个人信息保护法》《残疾人保障法》《无障碍环境建设法》等相关法律法规 |
| 技术栈 | 员工端采用跨平台框架（Flutter）开发以降低双端维护成本；辅导员端采用 Vue.js/React 前端框架 |

| 用户设备 | 部分员工可能使用中低端安卓手机，需确保在 3GB RAM 设备上流畅运行 |
